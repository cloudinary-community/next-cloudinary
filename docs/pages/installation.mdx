# Installing Next Cloudinary

* Install `next-cloudinary` with:
```
yarn add next-cloudinary
# or
npm install next-cloudinary
```

* Add an environment variable with your Cloud Name:
```
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME="<Your Cloud Name>"
```

The `CldUploadButton` and `CldUploadWidget` make use of an prop called `uploadPreset`. If new to upload presets, or are looking for guidance on how to edit/retrieve your existing preset(s), refer to this [guide](https://cloudinary.com/documentation/upload_presets#managing_upload_presets_using_the_settings_ui).

</Callout>

## CldImage

* Import the CldImage component:

```
import { CldImage } from 'next-cloudinary';
```

* Add the CldImage component pointing to a Cloudinary public ID:

```
<CldImage
  width="960"
  height="600"
  src="images/turtle"
  sizes="100vw"
  alt="Turtle in the ocean"
/>
```

<CldImage
  width="960"
  height="600"
  src={`${process.env.IMAGES_DIRECTORY}/turtle`}
  sizes="100vw"
  alt="Turtle in the ocean"
/>

## CldOgImage

* Import the CldOgImage component:

```
import { CldOgImage } from 'next-cloudinary';
```

* Add the CldOgImage component in your page outside of the Head component pointing to a Cloudinary public ID:

```
<CldOgImage
  src="images/galaxy"
  text={{
    text: 'Next Cloudinary',
    color: 'white'
  }}
/>
```

<Callout emoji={false}>
  CldOgImage does not render an `<img>` tag, meaning it can't be visually embedded on a page. The following examples make use of the `<CldImage>` tag to showcase what's possible.
</Callout>

  height={OG_IMAGE_HEIGHT}
