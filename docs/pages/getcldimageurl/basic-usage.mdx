# Getting Started with getCldImageUrl

You can use the getCldImageUrl helper function to generate Cloudinary URLs without the component wrapper. CldImage and other image-based components use getCldImageUrl or a derivitive of it to generate their images, meaning you can expect the same API from CldImage.

## Basic Usage

The basic required options include `width`, `height`, and `src`:

```js
import { getCldImageUrl } from 'next-cloudinary';

const url = getCldImageUrl({
  width: 960,
  height: 600,
  src: '<Public ID>'
});
```

<div style={{ maxWidth: 500, margin: '0 auto' }}>
  <CldImage
    src={getCldImageUrl({
      src: `${process.env.IMAGES_DIRECTORY}/galaxy`,
      width: 960,
      height: 600,
    })}
    width="960"
    height="600"
    sizes="(max-width: 480px) 100vw, 50vw"
    alt=""
    preserveTransformations
  />
</div>

<Callout emoji={false}>
  getCldImageUrl uses the same API as [CldImage](/cldimage/basic-usage). They're both wrappers around [@cloudinary-util/url-loader](https://github.com/colbyfayock/cloudinary-util/tree/main/packages/url-loader) which provide a simpler way to generate images and Cloudinary URLs.
</Callout>

## Transformations

You can further take advantage of Cloudinary features like background removal and overlays by adding additional props:

```js
const url = getCldImageUrl({
  width: 600,
  height: 600,
  src: '<Public ID>',
  crop: 'thumb',
  tint: '100:blue:green:red',
  blur: 1000,
});
```
